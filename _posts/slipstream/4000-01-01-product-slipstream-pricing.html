Pricing

Definitions

The following are terms used in the context of SlipStream, referred to when defining license capabilities and constraints.
  • Connector: A connector is an instance of a specific software component that allows a SlipStream server to interface with a specific IaaS cloud service, normally for a specific region and/or zone. For example, connectors are normally designed to point specifically to a single region for a given cloud. It is therefore standard practice to have several separate instances of the same connector (class) to interface SlipStream with several regions of the same cloud. License restrictions therefore normally refer to connector instance.
  • Open source connector: this refers to a connector released under an open source license (normally the same license as the SlipStream Server). SixSq® normally releases connectors which interface with open source IaaS implementations under an open source license.
  • Proprietary (closed source) connectors: this refers to connectors released under a proprietary license. SixSq® normally releases connectors which interface with a proprietary IaaS implementation or service under a proprietary license.
  • Active User: a user refers to an active user, as in the sense of SlipStream. Only active users can perform any type of action in the system.
  • Release: a release is identified using three numbers, dot separated. For example, 2.0.6.
  • Major release: a major release refers to the first number of the release string. For example the major release number for 2.0.6 is 2.
  • Minor release: a minor release refers to the second number of the release string. For example the minor release number for 2.0.6 is 0.
  • Release number: a release number refers to the third number of the release string. For example, the release number for 2.0.6 is 6.
  • Deployment: a deployment in the context of SlipStream is the creation of one virtual machine or several virtual machines deployed together.
  • Image creation: an image creation in the context of SlipStream is the creation of a new machine image, from an existing native image.
  • SixSq SlipStream SaaS: this app store is a SlipStream service managed by SixSq® and offered as a SaaS. This service is available as is, without any guarantees of availability. It holds a number of images and deployments, which can be used and exported to a private SlipStream instance.

Licensing Rules

The following licensing or subscription rules are applicable to SlipStream:

Connector

Each connector must be attached to a SlipStream Server bundle. For example, in the case where a customer already has reached the limit of the number of allowed connectors for his/her SlipStream Server license or subscription, if he/she wants to support an extra proprietary cloud, he/she will need to purchase a supplementary Extra Connector license or subscription, as well as a Proprietary Connector license or subscription. On the other hand, if the customer wants to support a new open source cloud, only the Extra Connector license or subscription is required. Each proprietary connector must be associated to a SlipStream Server license or subscription. In other words, a proprietary connector cannot be purchased alone. The license or subscription bearer is required to upgrade regularly, within the next full quarter, such that he/she always runs the latest version minus one minor version. If this constraint is not possible, a special maintenance and support contract is required.

Server Instances

A SlipStream license or subscription bearer gives right to a single instance of the service. Via its connector, however, it can federate IaaS clouds installed in a number of different sites.
